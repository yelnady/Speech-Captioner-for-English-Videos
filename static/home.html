Quick Project Pipeline

The stages of this wav2vec project.

1. Import

We imported libraries such as IPython.display, moviepy.editor, librosa, soundfile, transformers, numpy, and torch.

2. Convert

Using the functions from imported libraries, we can convert a video file or YouTube video into an audio file.

3. Generate

Using wav2vec 2.0 's pre-trained processor and model, we can generate subtitles for the given audio file.

Pre-trained Models & Fine Tuning

Creating a model from scratch would require a lot more time, computational power and data. Instead of making everything from scratch, we can use an existing model and then feed it less data in order to have it fit our needs better. By starting with a pre-trained model, you can fine-tune the parameters with new, specifically chosen datasets. Our choice of pre-trained model was Wav2Vec2-Base-960h.


The Dataset: Librispeech

Facebook's Wav2vec 2.​0 (960h)

The pre-trained models and code for Wav2vec 2.0 allows us to transcribe input speech audio with great accuracy. The base model we used in our speech recognition model was Wav2vec2-Base-960h, which was pre-trained and fine-tuned on 960 hours of Librispeech on 16kHz sampled speech audio. We were able to import it to our model through the transformers library.
